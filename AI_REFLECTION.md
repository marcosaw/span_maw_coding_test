# Approach to Generating the League Standing Table Solution

The solution is a clear, modular data‑processing pipeline that transforms raw match results into a complete league standing table. All input is a plain CSV file containing one match per line — `Date,HomeTeam,AwayTeam,HomeGoals,AwayGoals` — read either from standard input or from a filename passed on the command line. Each row is parsed into a `Match` record and validated (missing columns, non‑numeric or negative goals, a team "playing itself" all raise a clear, line‑numbered error rather than being silently skipped). The application then aggregates each team's played/won/drawn/lost, goals for/against, and points using the historically accurate two‑points‑for‑a‑win, one‑for‑a‑draw system of the 1974/75 English First Division. Standings are sorted by points, then by **goal average** (goals for ÷ goals against), then by goals for, then by team name — not goal difference. Goal difference wasn't adopted by the Football League until the 1976/77 season; using it for a 1974/75 table would be an anachronism, so goal average is the period‑accurate tie‑break here, and it's what the output's `GAvg` column reports.

The output is a text CSV in the conventional English league‑table format (`Pos,Team,P,W,D,L,F,A,GAvg,Pts`), one row per team, written to standard output or a specified output filename. The design is modular: `models.py` holds the data types and scoring rule, `io_csv.py` handles parsing/validation and output formatting, `standings.py` handles aggregation and sorting, and `cli.py` wires up argument parsing and the stdin/stdout‑vs‑file plumbing — each independently testable, which is exercised by a pytest suite covering parsing errors, the win/draw/loss and goal‑average tie‑break logic, and all four stdio/file CLI combinations. The submission includes both the input CSV (illustrative match results for the real 22 1974/75 First Division clubs through Week 10, since no reliably sourced round‑by‑round fixture list for that stretch of the season could be found) and the output CSV the program itself generates from it, satisfying the project's format and reproducibility requirements.
